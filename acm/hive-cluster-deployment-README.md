# Hive cluster deployment — disconnected OpenShift on AWS

This manifest deploys a self-managed OpenShift cluster into a **disconnected, proxy-based AWS environment** using **OpenShift Hive** and **Red Hat Advanced Cluster Management (RHACM)**. It is intended to run against the disconnected VPC infrastructure built by the `tf-rosa-platform` Terraform stack.

---

## What it deploys

Applying this manifest to an RHACM hub cluster creates the following resources:

| Resource | Kind | Purpose |
|---|---|---|
| `cluster3` | `Namespace` | Isolated namespace on the hub for all cluster resources |
| `cluster3` | `ClusterDeployment` | Instructs Hive to provision a new OCP cluster on AWS |
| `cluster3` | `ManagedCluster` | Registers the new cluster with RHACM for lifecycle management |
| `cluster3-worker` | `MachinePool` | Defines 3 worker nodes across 3 AZs (m5.xlarge, io1 100 GB) |
| `cluster3` | `KlusterletAddonConfig` | Enables RHACM add-ons (app manager, policy, search, cert policy) via proxy |
| `cluster3-install-config` | `Secret` | Hive install-config.yaml — cluster networking, proxy, mirror registry, subnets |
| `cluster3-pull-secret` | `Secret` | Registry auth pointing to the local Quay mirror registry |
| `cluster3-aws-creds` | `Secret` | AWS access key used by Hive to provision EC2/ELB/Route53 resources |
| `cluster3-ssh-private-key` | `Secret` | SSH private key for node access |
| `bound-service-account-signing-key` | `Secret` | Private key used to sign bound service account tokens (required for STS/Manual credentials mode) |
| `sts-manifests` | `Secret` | Pre-generated STS credential manifests for each operator (produced by `ccoctl`) |

### Key configuration points

**Disconnected / proxy setup** — the cluster has no direct internet access. All egress flows through the Squid proxy running on the bastion EC2. The `install-config.yaml` configures `httpProxy`, `httpsProxy`, and an explicit `noProxy` list covering all AWS VPC endpoints, the mirror registry, and the cluster's own base domain.

**Mirror registry** — OCP release images are pulled from the local Quay registry on the bastion rather than `quay.io`. Three `imageContentSources` entries redirect `quay.io/openshift-release-dev/ocp-release`, `quay.io/openshift-release-dev/ocp-v4.0-art-dev`, and `registry.redhat.io/multicluster-engine` to the local mirror.

**STS / Manual credentials mode** — `credentialsMode: Manual` is set in `install-config.yaml`. Each OCP operator uses a pre-generated AWS role ARN injected via the `sts-manifests` secret rather than long-lived static credentials.

**Internal publish** — `publish: Internal` means the cluster API and ingress are only reachable from within the VPC. There is no public-facing load balancer.

**RHACM integration** — the `ManagedCluster` and `KlusterletAddonConfig` resources register the new cluster back to the hub so it appears in the RHACM console immediately after provisioning completes.

---

## Prerequisites

- An RHACM hub cluster with the Hive operator installed.
- The `tf-rosa-platform` infrastructure stack already applied (disconnected VPC, subnets, proxy EC2 running).
- The `oc mirror` image sync completed on the bastion (Quay registry populated with the target OCP release).
- STS manifests generated by `ccoctl aws create-all` on the bastion.
- The Quay mirror registry root CA certificate available.
- A Red Hat pull secret merged with the local Quay registry credentials.
- An `ClusterImageSet` named `openshift-4-20-13` (or your chosen version) already applied to the hub.

## Variables

Copy the block below into a file named `vars.env`, fill in every value, then source it before running the generation script:

```bash
# ── Cluster identity ───────────────────────────────────────────────────────────
export CLUSTER_NAME="cluster3"
export BASE_DOMAIN="aws.sadiqueonline.com"
export AWS_REGION="ap-southeast-1"
export OCP_IMAGE_SET="openshift-4-20-13"

# ── AWS credentials (used by Hive to provision cloud resources) ────────────────
export AWS_ACCESS_KEY_ID="<aws_access_key_id>"
export AWS_SECRET_ACCESS_KEY="<aws_secret_access_key>"

# ── Networking — subnets from tf-disconnected outputs ─────────────────────────
# Run: terraform output disconnected_subnet_ids
export SUBNET_AZ1="subnet-080b1fe7e9714739f"
export SUBNET_AZ2="subnet-09486e56fc317ed1f"
export SUBNET_AZ3="subnet-05ddbd502f9ebbb5d"

# ── Proxy — bastion EC2 internal DNS name and port ────────────────────────────
# The bastion's internal hostname follows the pattern:
#   ip-<dashes-of-private-ip>.<region>.compute.internal
# Find it with: aws ec2 describe-instances --filters Name=tag:Name,Values=<cluster>-installer
export PROXY_INTERNAL_HOST="ip-172-17-1-66.ap-southeast-1.compute.internal"
export PROXY_PORT="3128"

# ── Mirror registry — Quay on the bastion ─────────────────────────────────────
export MIRROR_REGISTRY_HOST="ip-172-17-1-119.ap-southeast-1.compute.internal"
export MIRROR_REGISTRY_PORT="8443"
# base64 of "init:<mirror_registry_password>" — run: echo -n "init:PASSWORD" | base64
export MIRROR_REGISTRY_AUTH_B64="base64 encoded username:password"

# ── SSH keys ───────────────────────────────────────────────────────────────────
export SSH_PUBLIC_KEY="ssh-rsa AAAA..."
# Path to the SSH private key file
export SSH_PRIVATE_KEY_FILE="~/.ssh/id_rsa"

# ── STS / OIDC ─────────────────────────────────────────────────────────────────
# OIDC issuer URL — from: rosa describe oidc-config or terraform output
# This is the CloudFront URL generated when the OIDC config was created
export OIDC_ISSUER_URL="https://d378p33hvnfv3j.cloudfront.net"

# ── STS manifest files — generated by ccoctl on the bastion ───────────────────
# After running ccoctl, copy the manifests directory from the bastion:
#   scp -r ec2-user@<bastion_ip>:/home/ec2-user/sts/manifests ./sts-manifests/
export STS_MANIFESTS_DIR="./sts-manifests"

# ── Bound service account signing key ─────────────────────────────────────────
# Path to the private key file used to sign bound service account tokens
# Generated by ccoctl or openshift-install
export BOUND_SA_SIGNING_KEY_FILE="./sts/bound-service-account-signing-key.key"

# ── Quay root CA certificate ───────────────────────────────────────────────────
# Copy from the bastion: scp ec2-user@<bastion_ip>:/registry/quay-rootCA/rootCA.pem ./
export QUAY_ROOT_CA_FILE="./rootCA.pem"

# ── noProxy list ──────────────────────────────────────────────────────────────
# Extend this list if you add additional VPC endpoints or internal hostnames
export NO_PROXY="s3.dualstack.${AWS_REGION}.amazonaws.com,s3.${AWS_REGION}.amazonaws.com,ec2.${AWS_REGION}.amazonaws.com,elasticloadbalancing.${AWS_REGION}.amazonaws.com,sts.${AWS_REGION}.amazonaws.com,.registry.redhat.io,.quay.io,${MIRROR_REGISTRY_HOST},tagging.us-east-1.amazonaws.com,tagging.${AWS_REGION}.amazonaws.com,servicequotas.${AWS_REGION}.amazonaws.com,quay.io,registry.redhat.io,.${BASE_DOMAIN},route53.amazonaws.com,iam.amazonaws.com"
```

## Generate and apply the manifest

Once `vars.env` is populated, run the script below. It reads each variable and produces `output.yaml` ready to apply to the hub cluster.

```bash
#!/usr/bin/env bash
set -euo pipefail
source vars.env

# ── Derived values ─────────────────────────────────────────────────────────────
INSTALL_CONFIG_B64=$(cat <<ICEOF | base64 -w 0
apiVersion: v1
metadata:
  name: '${CLUSTER_NAME}'
baseDomain: ${BASE_DOMAIN}
credentialsMode: Manual
proxy:
  httpProxy: http://${PROXY_INTERNAL_HOST}:${PROXY_PORT}
  httpsProxy: http://${PROXY_INTERNAL_HOST}:${PROXY_PORT}
  noProxy: ${NO_PROXY}
controlPlane:
  architecture: amd64
  hyperthreading: Enabled
  name: master
  replicas: 3
  platform:
    aws:
      rootVolume:
        iops: 4000
        size: 100
        type: io1
      type: m5.xlarge
compute:
- hyperthreading: Enabled
  architecture: amd64
  name: 'worker'
  replicas: 3
  platform:
    aws:
      rootVolume:
        iops: 2000
        size: 100
        type: io1
      type: m5.xlarge
networking:
  networkType: OVNKubernetes
  clusterNetwork:
  - cidr: 10.128.0.0/14
    hostPrefix: 23
  machineNetwork:
  - cidr: 172.16.0.0/16
  serviceNetwork:
  - 172.30.0.0/16
platform:
  aws:
    region: ${AWS_REGION}
    zones:
      - ${AWS_REGION}a
      - ${AWS_REGION}b
      - ${AWS_REGION}c
    subnets:
      - ${SUBNET_AZ1}
      - ${SUBNET_AZ2}
      - ${SUBNET_AZ3}
    serviceEndpoints:
      - name: sts
        url: https://sts.${AWS_REGION}.amazonaws.com
publish: Internal
imageContentSources:
- mirrors:
  - ${MIRROR_REGISTRY_HOST}:${MIRROR_REGISTRY_PORT}/openshift/release-images
  source: quay.io/openshift-release-dev/ocp-release
- mirrors:
  - ${MIRROR_REGISTRY_HOST}:${MIRROR_REGISTRY_PORT}/openshift/release
  source: quay.io/openshift-release-dev/ocp-v4.0-art-dev
- mirrors:
  - ${MIRROR_REGISTRY_HOST}:${MIRROR_REGISTRY_PORT}/multicluster-engine
  source: registry.redhat.io/multicluster-engine
pullSecret: "" # skip, hive will inject based on it's secrets
sshKey: |-
    ${SSH_PUBLIC_KEY}
additionalTrustBundle: |-
$(sed 's/^/    /' "${QUAY_ROOT_CA_FILE}")
ICEOF
)

BOUND_SA_KEY_B64=$(base64 -w 0 < "${BOUND_SA_SIGNING_KEY_FILE}")
SSH_PRIVATE_KEY_B64=$(base64 -w 0 < "${SSH_PRIVATE_KEY_FILE}")

# Build sts-manifests secret data block from all files in the directory
STS_DATA_BLOCK=""
for f in "${STS_MANIFESTS_DIR}"/*.yaml; do
  key=$(basename "$f")
  val=$(base64 -w 0 < "$f")
  STS_DATA_BLOCK="${STS_DATA_BLOCK}  ${key}: ${val}"$'\n'
done

# Build cluster-authentication-02-config.yaml entry for sts-manifests
CLUSTER_AUTH_B64=$(cat <<AUTHEOF | base64 -w 0
apiVersion: config.openshift.io/v1
kind: Authentication
metadata:
  name: cluster
spec:
  serviceAccountIssuer: ${OIDC_ISSUER_URL}
AUTHEOF
)

cat <<EOF > output.yaml
apiVersion: v1
kind: Namespace
metadata:
  name: ${CLUSTER_NAME}
---
apiVersion: hive.openshift.io/v1
kind: ClusterDeployment
metadata:
  name: ${CLUSTER_NAME}
  namespace: ${CLUSTER_NAME}
  labels:
    cloud: AWS
    region: ${AWS_REGION}
    vendor: OpenShift
    cluster.open-cluster-management.io/clusterset: default
spec:
  boundServiceAccountSigningKeySecretRef:
    name: bound-service-account-signing-key
  baseDomain: ${BASE_DOMAIN}
  clusterName: ${CLUSTER_NAME}
  controlPlaneConfig:
    servingCertificates: {}
  installAttemptsLimit: 1
  installed: false
  platform:
    aws:
      credentialsSecretRef:
        name: ${CLUSTER_NAME}-aws-creds
      region: ${AWS_REGION}
      privateLink:
        enabled: false
  provisioning:
    manifestsSecretRef:
      name: sts-manifests
    installConfigSecretRef:
      name: ${CLUSTER_NAME}-install-config
    sshPrivateKeySecretRef:
      name: ${CLUSTER_NAME}-ssh-private-key
    imageSetRef:
      name: ${OCP_IMAGE_SET}
  pullSecretRef:
    name: ${CLUSTER_NAME}-pull-secret
---
apiVersion: cluster.open-cluster-management.io/v1
kind: ManagedCluster
metadata:
  name: ${CLUSTER_NAME}
  labels:
    name: ${CLUSTER_NAME}
    cloud: Amazon
    region: ${AWS_REGION}
    vendor: OpenShift
    cluster.open-cluster-management.io/clusterset: default
spec:
  hubAcceptsClient: true
---
apiVersion: hive.openshift.io/v1
kind: MachinePool
metadata:
  name: ${CLUSTER_NAME}-worker
  namespace: ${CLUSTER_NAME}
spec:
  name: worker
  clusterDeploymentRef:
    name: ${CLUSTER_NAME}
  platform:
    aws:
      zones:
        - ${AWS_REGION}a
        - ${AWS_REGION}b
        - ${AWS_REGION}c
      rootVolume:
        iops: 2000
        size: 100
        type: io1
      type: m5.xlarge
  replicas: 3
---
apiVersion: v1
kind: Secret
metadata:
  name: ${CLUSTER_NAME}-pull-secret
  namespace: ${CLUSTER_NAME}
stringData:
  .dockerconfigjson: |
    {
      "auths": {
        "${MIRROR_REGISTRY_HOST}:${MIRROR_REGISTRY_PORT}": {
          "auth": "${MIRROR_REGISTRY_AUTH_B64}"
        }
      }
    }
type: kubernetes.io/dockerconfigjson
---
apiVersion: v1
kind: Secret
metadata:
  name: ${CLUSTER_NAME}-install-config
  namespace: ${CLUSTER_NAME}
type: Opaque
data:
  install-config.yaml: ${INSTALL_CONFIG_B64}
---
apiVersion: v1
kind: Secret
metadata:
  name: ${CLUSTER_NAME}-ssh-private-key
  namespace: ${CLUSTER_NAME}
type: Opaque
data:
  ssh-privatekey: ${SSH_PRIVATE_KEY_B64}
---
apiVersion: v1
kind: Secret
type: Opaque
metadata:
  name: ${CLUSTER_NAME}-aws-creds
  namespace: ${CLUSTER_NAME}
stringData:
  aws_access_key_id: ${AWS_ACCESS_KEY_ID}
  aws_secret_access_key: ${AWS_SECRET_ACCESS_KEY}
---
apiVersion: agent.open-cluster-management.io/v1
kind: KlusterletAddonConfig
metadata:
  name: ${CLUSTER_NAME}
  namespace: ${CLUSTER_NAME}
spec:
  clusterName: ${CLUSTER_NAME}
  clusterNamespace: ${CLUSTER_NAME}
  clusterLabels:
    cloud: Amazon
    vendor: OpenShift
  applicationManager:
    proxyPolicy: OCPGlobalProxy
    enabled: true
  policyController:
    proxyPolicy: OCPGlobalProxy
    enabled: true
  searchCollector:
    proxyPolicy: OCPGlobalProxy
    enabled: true
  certPolicyController:
    proxyPolicy: OCPGlobalProxy
    enabled: true
---
apiVersion: v1
kind: Secret
metadata:
  name: bound-service-account-signing-key
  namespace: ${CLUSTER_NAME}
type: Opaque
data:
  bound-service-account-signing-key.key: ${BOUND_SA_KEY_B64}
---
apiVersion: v1
kind: Secret
metadata:
  name: sts-manifests
  namespace: ${CLUSTER_NAME}
data:
  cluster-authentication-02-config.yaml: ${CLUSTER_AUTH_B64}
${STS_DATA_BLOCK}
EOF

echo "output.yaml generated successfully."
echo "Review it, then apply: oc apply -f output.yaml"
```

Save the script as `generate.sh`, make it executable, and run it:

```bash
chmod +x generate.sh
source vars.env
./generate.sh
```

Then apply to the hub cluster:

```bash
oc apply -f output.yaml
```

## Monitoring the installation

Watch provisioning progress from the hub:

```bash
# Overall ClusterDeployment status
oc get clusterdeployment ${CLUSTER_NAME} -n ${CLUSTER_NAME} -w

# Live Hive installer pod logs
oc logs -f -n ${CLUSTER_NAME} \
  $(oc get pods -n ${CLUSTER_NAME} -l hive.openshift.io/job-type=provision \
    -o jsonpath='{.items[0].metadata.name}') \
  -c hive

# Check ManagedCluster registration
oc get managedcluster ${CLUSTER_NAME}
```

Installation typically takes 30–45 minutes. When complete, `ClusterDeployment.spec.installed` becomes `true` and the cluster appears as `Available` in the RHACM console.

## Retrieve the kubeconfig

```bash
# Get the admin kubeconfig secret name
oc get clusterdeployment ${CLUSTER_NAME} -n ${CLUSTER_NAME} \
  -o jsonpath='{.spec.clusterMetadata.adminKubeconfigSecretRef.name}'

# Extract the kubeconfig
oc get secret -n ${CLUSTER_NAME} \
  $(oc get clusterdeployment ${CLUSTER_NAME} -n ${CLUSTER_NAME} \
    -o jsonpath='{.spec.clusterMetadata.adminKubeconfigSecretRef.name}') \
  -o jsonpath='{.data.kubeconfig}' | base64 -d > cluster3-kubeconfig

export KUBECONFIG=./cluster3-kubeconfig
oc get nodes
```

## Teardown

```bash
# Delete the ClusterDeployment — Hive will deprovision all AWS resources first
oc delete clusterdeployment ${CLUSTER_NAME} -n ${CLUSTER_NAME}

# Watch deprovision progress
oc get clusterdeployment ${CLUSTER_NAME} -n ${CLUSTER_NAME} -w

# Once gone, clean up the namespace
oc delete namespace ${CLUSTER_NAME}
```

> **Note:** Do not delete the namespace before the `ClusterDeployment` is fully deprovisioned, otherwise the AWS resources (EC2 instances, ELBs, Route53 records) will be orphaned.
